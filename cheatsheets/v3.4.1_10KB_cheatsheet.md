Useful terms:

- rightmost variable = last variable appearing in the full leaf-reading of a term

- PARITY(term) = for each variable, whether it appears an even or odd number of times
- same WORD on the two sides of an equation means the full leaf lists are exactly equal
- same PARITY on the two sides of an equation means every variable has the same parity count on both sides

STEP 2. IMMEDIATE EXACT TRUE ROUTES

2.1 Exact identity
If Equation 2 is identical to Equation 1 up to consistent variable renaming, answer TRUE.
RULE: exact renaming

2.2 Tautological target
If Equation 2 is of the form x = x, answer TRUE.
RULE: tautological target

2.3 Singleton / collapse source
If Equation 1 has the form x = T where x does not appear anywhere in T, then Equation 1 forces a singleton magma. Therefore every equation holds. Answer TRUE.
Examples:

- x = y
- x = y\*z
- x = (y*z)*w
- x = y*(z*w)
  RULE: singleton collapse

  2.4 Stronger singleton variants
  If Equation 1 clearly says that one variable equals a term built only from other variables, even after consistent renaming or side-swap, it is still singleton collapse.
  Example:

- (y*z)*w = x
  RULE: singleton collapse

  2.5 Constant-operation source
  If Equation 1 has the form A = B where:

- both A and B are non-variable product terms, and
- the variable sets of A and B are disjoint,
  then Equation 1 forces all products to be equal to one constant value. In that case every equation built only from products is true, and every equation x = product is also true because x can be chosen as that constant. Answer TRUE.
  Typical examples:
- x*y = z*w
- (x*y)*z = u\*v
  RULE: constant operation

  2.6 Left projection family
  If Equation 1 has one of these shapes, then it forces a left-projection style behavior strong enough for many targets:

- x = x\*y
- x = x*(y*z)
- x = x*(y*(z\*w))
- x = (x*y)*x
- x = ((x*y)*z)\*x
  When Equation 2 is obtained by inserting extra factors to the right of x, or by replacing a right-side subterm with another right-side product while preserving the leading x, answer TRUE.
  Typical examples:
- x = x*y implies x = x*(u\*v)
- x = (y*z)*x implies x = ((u*v)*w)\*x
  RULE: projection family

  2.7 Right projection family
  Dual of 2.6. If Equation 1 is of the form:

- x = y\*x
- x = (y*z)*x
- x = y*(z*x)
  and Equation 2 preserves the trailing x while adding or reshaping material on the left, answer TRUE.
  RULE: projection family

  2.8 Square-reduction family
  If Equation 1 has one of these forms:

- x\*x = x
- x*(x*x) = x
- (x*x)*x = x
  then any target obtained by replacing a repeated x-block by x is TRUE.
  Examples:
- x*(x*x) = x implies x*(x*(x\*x)) = x
- (x*x)*x = x implies ((x*x)*x)*z = x*z
  RULE: square reduction

  2.9 Source stronger than target by direct contextual instance
  If Equation 2 is exactly a substitution instance of Equation 1, or exactly a one-hole context instance of Equation 1, answer TRUE.
  Examples:

- Source: x = y\*x
- Target: (z*x) = z*(y\*x) is NOT allowed unless it is an exact one-hole context instance
- Target: (u*x)*v = (u*(y*x))\*v is allowed if it is exact
  RULE: exact instance

STEP 3. CLEAR SYNTACTIC FALSE ROUTES
These are valid only when their literal premise is satisfied exactly on the written equations.
Do not use them approximately.

3.1 WORD exclusion
If the two sides of Equation 1 have exactly the same full WORD, but the two sides of Equation 2 have different WORD, answer FALSE.
Reason: equality of WORD is preserved in every semigroup / concatenation model satisfying Equation 1.
Legal example:

- Eq1: x*(y*z) = (x*y)*z has WORD x y z on both sides
- Eq2: x*(y*z) = y*(x*z) has WORD x y z vs y x z, so FALSE
  Illegal example:
- Eq1: x = y*(z*w) does NOT have same WORD on both sides, so this rule may not be used
  RULE: word exclusion

  3.2 PARITY exclusion
  If the two sides of Equation 1 have exactly the same variable parity pattern, but the two sides of Equation 2 have different parity pattern, answer FALSE.
  Reason: parity is preserved in XOR-like linear finite models.
  Legal example:

- Eq1: x = (y*x)*y gives x-parity odd on both sides and y-parity even on both sides
- Eq2: x = y\*x gives y-parity 0 vs 1, so FALSE
  RULE: parity exclusion

  3.3 Leftmost-variable exclusion
  If Equation 1 has the same leftmost variable on both sides, but Equation 2 has different leftmost variables on its two sides, answer FALSE.
  Use only when this is literally true after allowed normalization.
  RULE: leftmost exclusion

  3.4 Rightmost-variable exclusion
  If Equation 1 has the same rightmost variable on both sides, but Equation 2 has different rightmost variables on its two sides, answer FALSE.
  Use only when this is literally true after allowed normalization.
  RULE: rightmost exclusion

STEP 4. SMALL FINITE WITNESS ROUTES
Use only very standard finite magmas. A counterexample is decisive only if Equation 1 holds for ALL assignments and Equation 2 fails for SOME assignment.
Never claim a witness unless you checked both conditions.

4.1 Left-zero magma
Operation: a\*b = a
If Equation 1 is valid in left-zero but Equation 2 is not, answer FALSE.
In left-zero every term evaluates to its leftmost variable.
So:

- Eq1 holds iff its two sides have the same leftmost variable
- Eq2 fails iff its two sides have different leftmost variables
  RULE: left-zero witness

  4.2 Right-zero magma
  Operation: a\*b = b
  If Equation 1 is valid in right-zero but Equation 2 is not, answer FALSE.
  In right-zero every term evaluates to its rightmost variable.
  RULE: right-zero witness

  4.3 XOR / parity witness on {0,1}
  Operation: a\*b = a+b mod 2
  If Equation 1 is valid because both sides have the same parity pattern, but Equation 2 fails because parity differs, answer FALSE.
  RULE: XOR parity witness

  4.4 Constant witness on {0,1}
  Operation: a*b = 0 for all a,b, or a*b = 1 for all a,b
  Use only if Equation 1 is valid under that constant operation for all assignments and Equation 2 is not.
  Do not guess. Check literally.
  RULE: constant witness
